Remarks

A setting service is used to load, save, and manage settings.

Settings are loaded from and saved to storages (implementing ISettingStorage). How a setting is stored depends on the ISettingStorage implementation. However, the values written to and loaded from a storage are strings. The storage is responsible to convert them into the format used by the storage.

The conversion from/to strings used by the storages is done using converters (implementing ISettingConverter). The converters allow conversion between the storage and actual types (e.g. int from/to string). A converter must always provide in both directions.

If a type is supported by multiple converters, the one which has the StringConversion conversion mode is used. Otherwise, one is used but it is not specified which one.

A storage can be read-only (see IsReadOnly). Values are persisted with all available non-read-only storages and therefore are saved multiple times if multiple non-read-only storages are available.

When values are filtered based on their names using WriteOnlyKnown. In such cases, a storage only writes values whose names are already available in the storage.

When values are filtered based on their names using WritePrefixAffinities. In such cases, a storage only writes values whose names start with the specified prefix.

When retrieving values, values from read-only storages have higher priority if they have the same setting name. Otherwise, the order is undefined for values with the same name.

Names of settings are considered case-insensitive.

Values are not automatically read from the storages. Load must be called to do so. Values are not automatically written to the storages. Save must be called to do so.
